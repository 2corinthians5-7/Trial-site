GitHub Pages Redirect Fix

How to make your GitHub Pages site display your HTML file instead of README.md

Understanding the Problem

By default, GitHub Pages serves index.html as the default file for your site. However, if you don't have an index.html file in your root directory, it may fall back to showing your README.md file instead.

This happens because GitHub prioritizes certain filenames when serving content. The order of priority is typically:

  1. index.html
  2. index.md
  3. README.md

Solutions

Here are several methods to fix this issue:

1 Rename your HTML file to index.html

The simplest solution is to rename your main HTML file to index.html and place it in the root directory of your repository.

mv your-file.html index.html

Note: This is the most reliable method and follows web standards for default documents.

2 Use a custom index.html that redirects

If you can't rename your main HTML file, create an index.html file that redirects to your desired HTML file:

<!DOCTYPE html>
<html>
<head>
    <meta charset="utf-8">
    <title>Redirecting...</title>
    <meta http-equiv="refresh" content="0; URL=your-file.html">
</head>
<body>
    <p>Redirecting to <a href="your-file.html">your-file.html</a></p>
</body>
</html>

3 Configure with _config.yml (Jekyll)

If you're using Jekyll (GitHub Pages' default site generator), you can configure the default page in your _config.yml file:

# _config.yml
permalink: /:title/
defaults:
  - scope:
      path: ""
    values:
      layout: "default"
      permalink: /your-file.html

Note: This method requires understanding of Jekyll configuration.

4 Use GitHub Pages settings

If your site is built from the /docs folder or a different branch, make sure your settings are configured correctly:

  1. Go to your GitHub repository
  2. Click on "Settings"
  3. Scroll down to "GitHub Pages" section
  4. Ensure the source is set correctly (e.g., main branch, /docs folder)

Recommended Folder Structure

your-repository/
├── index.html   # Your main HTML file (renamed)
├── css/
│ └── style.css
├── js/
│ └── script.js
├── images/
│ └── ...
└── README.md   # Your README file

Success Tip: After implementing any of these solutions, it might take a few minutes for GitHub Pages to update. You can force a refresh by making a minor change to your repository or waiting up to 10 minutes.
